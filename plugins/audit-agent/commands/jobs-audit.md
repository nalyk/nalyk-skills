## THE 13 QUESTIONS

1. **SIMPLIFICATION** — "How can I make this simpler?" Strip away everything unnecessary; what can be removed without losing core value?
2. **ZERO-BASED THINKING** — "What would this look like if I started from zero?" Ignore existing constraints and legacy decisions; design from scratch.
3. **CORE FUNCTION** — "What's the ONE thing this absolutely must do perfectly?" Everything else is secondary.
4. **BEGINNER'S MIND** — "How would I design this for someone who's never seen it before?" Eliminate insider assumptions; what confuses a newcomer?
5. **ELEGANCE** — "What would the most elegant solution be?" Where does form meet function perfectly?
6. **COMPLEXITY AUDIT** — "Where am I adding complexity users don't value?" What exists because you CAN, not because you SHOULD?
7. **MAGIC EXPERIENCE** — "What would this be like if it just worked magically?" Design the invisible interface.
8. **INSANELY GREAT** — "How would I make this insanely great instead of just good?" What would make users say "wow"?
9. **RESTRAINT CHECK** — "What am I including because I can, not because I should?" What should be cut even though it's possible?
10. **ACCESSIBLE SIMPLICITY** — "How can I make the complex appear simple?" What metaphor or abstraction makes this intuitive?
11. **PERSONAL USE CASE** — "What would this look like if I designed it for myself?" What would YOU actually want to use?
12. **QUALITY BLIND SPOTS** — "Where am I compromising that I shouldn't be?" Where did "good enough" sneak in?
13. **INEVITABLE FLOW** — "How can I make this feel inevitable instead of complicated?" What makes users feel guided, not confused?
